Purpose for OSI model:

Think about it. If one day you want to tell someone that there is a problem with your network connection. What are you going to say?
“Hey, there is a problem in connecting the server! ” (what kind of problem? )
“Hey, the server is not responding!” (what do you mean “not responding?” )
When network engineers communicate these types of questions often lead to misinterpretation. So, what is a reasonable way to name a specific type of error? If you are looking for an answer to this question, OSI model is what you’re searching for. OSI model guides the vendors and developers so that the program they’ve been using can interoperate (meaning that diverse communication system can connect and communicate; this is just a brief explanation, Strongly suggest you do some research)

What is OSI model:

OSI model stands for Open Systems Interconnection model, which is a conceptual framework created by the International Organization of Standardization. It is designed to characterize communication functions of diverse computing systems by dividing network communication into 7 abstract layers: Physical layer, Data Link layer, Network layer, Transport layer, Session layer, Presentation layer, and Application layer.

Data Link Layer

When you think of Data Link layer, consider it as the manager who controls data to be transferred from point to point. However, it is only in charge of the data that transfer under the same network. What is interesting is that Data Link layer takes packages from the network layer and breaks them down into frames. Let's now talk about the major functions of Data Link layer.

    Framing:
    As I've mentioned, Data Link Layer will break stream of bits that is received from Network Layer, into a manageable unit called frame.
    Physical Addressing:
    Data Link layer will add a header in front of the frame, making it recognizable if it is sended to different systems on the network.
    Flow Control:
    What if all cars on a highway drive at different speeds? Probably there will be a serious traffic jam. Just like how network will stuck if a fast transmitter connects with a slow receiver. Data Link layer solves this porblem, making devices connect fluently by controlling the flow.
    Error Control:
    To prevent losing frames and unintentional duplications, Data Link layer adds a trailer at the end of the frame to detect if anything is missing.
    Multiple Access:
    To prevent collisions, Data Link layer provides mechanisms such as CSMA/CD to make it possible to access shared media from different systems. If you are wondering what CSMA/CD is, here is a link for you to learn more.
